# FitTrack 仕様書

## 1. アプリ概要

| 項目 | 内容 |
|:---|:---|
| アプリ名 | FitTrack |
| プラットフォーム | Android |
| 対応バージョン | Android 8.0（API 26）以上 |
| ターゲットSDK | 35 |
| 言語 | 日本語 |
| 価格 | 無料（広告なし・サブスクなし） |
| ネットワーク | 不要（オフライン完全対応） |

FitTrackは、ジムでのトレーニング記録をストレスなく残し、成長を可視化するためのAndroidアプリである。ノートに書く感覚で使えるシンプルさを最優先とし、余計な機能を排除した設計となっている。

## 2. ターゲットユーザー

筋トレ中級者から上級者（週3〜5回ジム通い）を主なターゲットとする。トレーニング中にスマートフォンを操作する時間を最小限に抑え、記録の正確性と振り返りの効率を重視するユーザーを想定している。

## 3. 技術スタック

| レイヤー | 技術 |
|:---|:---|
| 言語 | Kotlin |
| UIフレームワーク | Jetpack Compose |
| ナビゲーション | Navigation Compose |
| ローカルDB | Room（SQLite） |
| 状態管理 | ViewModel + StateFlow |
| DI | Hilt（Dagger） |
| グラフ描画 | Vico |
| 設定保存 | SharedPreferences |
| ビルドシステム | Gradle（Kotlin DSL） |

## 4. 画面一覧

アプリは以下の7画面で構成される。ボトムナビゲーションバーにより、ホーム・履歴・統計・設定の4画面を切り替える。

| 画面 | ルート | ナビゲーション |
|:---|:---|:---|
| ホーム | `home` | ボトムナビ |
| セッション記録 | `session/{sessionId}` | ホームから遷移 |
| エクササイズ選択 | `exercise_select/{sessionId}` | セッション記録から遷移 |
| 履歴一覧 | `history` | ボトムナビ |
| 履歴詳細 | `history_detail/{sessionId}` | 履歴一覧から遷移 |
| 統計 | `stats` | ボトムナビ |
| 設定 | `settings` | ボトムナビ |

## 5. 画面仕様

### 5.1 ホーム画面

ホーム画面はアプリ起動時に最初に表示される画面であり、トレーニングの開始と直近の記録確認を主な目的とする。

**表示要素:**

- 今日の日付
- 「トレーニング開始」ボタン（メインCTA）
- 最近のトレーニング履歴サマリー（直近5件）
  - 日付、種目数、セット数、総ボリューム（kg）
- 進行中のセッションがある場合は「再開」ボタンを表示

**操作:**

- 「トレーニング開始」をタップすると新規セッションが作成され、セッション記録画面に遷移する
- 履歴サマリーのカードをタップすると履歴詳細画面に遷移する

### 5.2 セッション記録画面

ジムで実際に使用するメイン画面である。操作タップ数を最小化し、片手操作を前提とした設計となっている。

**表示要素:**

- トップバーに「トレーニング記録」タイトルと「終了」ボタン
- エクササイズ別のカード
  - エクササイズ名（太字・大きめ表示）
  - 「前回を見る」ボタン（タップで前回記録を展開表示）
  - セット一覧（SET番号・重量入力・回数入力・完了ボタン）
  - 「セット追加」ボタン
- FAB「エクササイズ追加」ボタン

**操作:**

- 重量入力は数値キーボード（小数対応）
- 回数入力は数値キーボード
- セット完了ボタンをタップすると記録が確定される
- 新しいセット追加時、前のセットの重量が自動引き継ぎされる
- 「終了」をタップするとセッションの終了時刻が記録され、ホームに戻る

### 5.3 エクササイズ選択画面

セッション記録画面からエクササイズを追加する際に表示される。

**表示要素:**

- 検索バー（テキスト入力によるフィルタリング）
- カテゴリフィルターチップ（すべて・胸・背中・肩・腕・脚・体幹・有酸素）
- カテゴリ別にグループ化されたエクササイズ一覧

**操作:**

- フィルターチップをタップするとカテゴリで絞り込み
- 検索バーに入力するとエクササイズ名で検索
- エクササイズをタップすると選択され、セッション記録画面に戻る（最初のセットが自動追加される）

### 5.4 履歴一覧画面

過去のトレーニングセッションを日付順に一覧表示する。

**表示要素:**

- セッションカード一覧（新しい順）
  - 日付（例：2026年3月29日（日））
  - 種目数・セット数・総ボリューム
  - 削除ボタン

**操作:**

- カードをタップすると履歴詳細画面に遷移する
- 削除ボタンをタップすると確認ダイアログが表示され、承認後に削除される

### 5.5 履歴詳細画面

特定のセッションの全記録を表示する。

**表示要素:**

- セッション情報カード（日付・開始時刻・終了時刻・種目数）
- エクササイズ別カード
  - エクササイズ名・対象筋群
  - セット一覧（SET番号・重量・回数・完了状態）

### 5.6 統計画面

トレーニングの成長を可視化する画面である。

**表示要素:**

- エクササイズ選択ドロップダウン
- 選択したエクササイズの最大重量推移グラフ（折れ線グラフ、Vico使用）
- 自己ベスト一覧（全エクササイズの最大重量と記録日）

**操作:**

- ドロップダウンでエクササイズを切り替えるとグラフが更新される

### 5.7 設定画面

アプリの表示設定を管理する。

**表示要素:**

- ダークモード設定（ラジオボタン：システム追従・ライト・ダーク）
- 重量単位設定（フィルターチップ：kg・lbs）
- アプリ情報（バージョン・ビルド番号）

## 6. データモデル

### 6.1 Exercise（エクササイズマスター）

| カラム | 型 | 説明 |
|:---|:---|:---|
| id | Long | 主キー（自動採番） |
| name | String | エクササイズ名 |
| category | ExerciseCategory | カテゴリ（CHEST, BACK, SHOULDER, ARM, LEG, CORE, CARDIO） |
| muscleGroup | String | 対象筋群（表示用テキスト） |
| isCustom | Boolean | ユーザー追加フラグ |

### 6.2 TrainingSession（トレーニングセッション）

| カラム | 型 | 説明 |
|:---|:---|:---|
| id | Long | 主キー（自動採番） |
| date | Long | セッション日付（エポックミリ秒） |
| startTime | Long | 開始時刻（エポックミリ秒） |
| endTime | Long? | 終了時刻（null = 進行中） |
| note | String | メモ |

### 6.3 ExerciseSet（エクササイズセット）

| カラム | 型 | 説明 |
|:---|:---|:---|
| id | Long | 主キー（自動採番） |
| sessionId | Long | 外部キー → TrainingSession.id |
| exerciseId | Long | 外部キー → Exercise.id |
| setNumber | Int | セット番号 |
| weightKg | Float | 重量（kg） |
| reps | Int | 回数 |
| isCompleted | Boolean | 完了フラグ |
| note | String | セット別メモ |

### 6.4 リレーション

```
TrainingSession 1 ─── * ExerciseSet * ─── 1 Exercise
```

一つのセッションは複数のセットを持ち、各セットは一つのエクササイズに紐づく。

## 7. プリセットエクササイズ

アプリ初回起動時にRoomのコールバックで以下のエクササイズが自動登録される。

| カテゴリ | エクササイズ |
|:---|:---|
| 胸 | ベンチプレス、インクラインベンチプレス、ダンベルフライ、チェストプレス、プッシュアップ |
| 背中 | デッドリフト、ラットプルダウン、ベントオーバーロウ、シーテッドロウ、チンアップ、懸垂 |
| 肩 | ショルダープレス、サイドレイズ、フロントレイズ、リアデルトフライ |
| 腕 | バーベルカール、ハンマーカール、トライセプスプッシュダウン、ディップス、スカルクラッシャー |
| 脚 | スクワット、レッグプレス、ルーマニアンデッドリフト、レッグカール、レッグエクステンション、カーフレイズ、ランジ |
| 体幹 | プランク、クランチ、レッグレイズ、ロシアンツイスト |
| 有酸素 | ランニング、サイクリング、ロープ跳び、ウォーキング |

## 8. ナビゲーション構造

```
BottomNavigation
├── ホーム (home)
│   ├── → セッション記録 (session/{id})
│   │   └── → エクササイズ選択 (exercise_select/{id})
│   └── → 履歴詳細 (history_detail/{id})
├── 履歴 (history)
│   └── → 履歴詳細 (history_detail/{id})
├── 統計 (stats)
└── 設定 (settings)
```

ボトムナビゲーション間の遷移ではバックスタックが保存され、各タブの状態が維持される。セッション記録やエクササイズ選択などのサブ画面ではボトムナビゲーションバーは非表示となる。

## 9. デザイン方針

- **Material Design 3** 準拠
- **ダイナミックカラー** 対応（Android 12以上）、非対応端末ではBlue系のフォールバックカラーを使用
- ダークモード・ライトモードの両方をサポート
- タップ領域は最低44dp確保
- 数値入力フィールドは大きめに設計し、ジムでの操作性を重視
- アニメーションは最小限に抑え、パフォーマンスを優先

## 11. ビルド・実行

### 前提条件

- Android Studio Ladybug（2024.2.1）以降
- JDK 17
- Android SDK 35

### デバッグビルド

```bash
./gradlew assembleDebug
```

APKは `app/build/outputs/apk/debug/app-debug.apk` に生成される。

### リリースビルド

```bash
./gradlew assembleRelease
```

リリースビルドには署名設定が別途必要となる。
